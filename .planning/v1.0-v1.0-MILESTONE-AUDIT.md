## Failed/Gaps Found

### Critical: Orchestrator GovernanceClient Interface Mismatch

**Problem:** The orchestrator (`src/orchestrator/executor.ts`) defines its own minimal `GovernanceClient` interface:
```typescript
interface GovernanceClient {
  checkPolicy(channelId: string, action: string): Promise<GovernanceCheck>;
  checkBudget(sessionId: string, action: string): Promise<GovernanceCheck>;
}
```

But the actual `GovernanceClient` class (`src/governance/client.ts`) provides:
```typescript
class GovernanceClient {
  evaluateToolRequest(request: ToolRequestContext): PolicyDecision
  requestApproval(request, decision): Promise<ApprovalEntry | null>
  // + telemetry methods
}
```

**Impact:** `setGovernanceClient()` in executor is **never called in production code** — only in tests. This means task-level policy enforcement in the orchestrator is non-functional.

**Evidence:**
```bash
grep -r "setGovernanceClient\(" src/ --include="*.ts" | grep -v test
# Returns only: src/orchestrator/executor.ts:87 (the definition itself)
```
