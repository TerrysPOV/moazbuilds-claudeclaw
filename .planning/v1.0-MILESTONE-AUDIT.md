## Critical Gaps

### 1. Human Escalation Unwired (CRITICAL)

**Problem:** Phase 6 built pause/resume, handoff, notifications, triggers — but none are actually used.

**Evidence:**
- `shouldBlockAdmission()` exported from `escalation/pause.ts` — never called
- `shouldBlockScheduling()` exported — never called  
- `handleEscalationTrigger()` available — never called
- No imports from `escalation/` in `gateway/` or `orchestrator/` directories

**Impact:**
- Operators cannot actually pause the daemon
- Escalation notifications never fire
- Handoff packages can be created but don't auto-trigger on failures

**Fix Required:**
```typescript
// In gateway/index.ts
import { shouldBlockAdmission } from '../escalation/pause';

// Before processing event:
if (shouldBlockAdmission()) {
  // Reject or defer event
}
```

```typescript
// In orchestrator/executor.ts
import { shouldBlockScheduling } from '../escalation/pause';

// Before executing tasks:
if (shouldBlockScheduling()) {
  // Defer task execution
}
```

### 2. Escalation Triggers Unwired (CRITICAL)

**Problem:** Triggers.ts has handlers for policy denials, watchdog, DLQ, orchestration failures — but nothing calls them.

**Evidence:**
- `handlePolicyDenial()` — never called from policy engine
- `handleWatchdogTrigger()` — never called from runner
- `handleDlqOverflow()` — never called from DLQ
- `handleOrchestrationFailure()` — never called from executor

**Impact:**
- Critical events don't generate notifications
- Operators aren't alerted to system issues
- Manual escalation only (no automatic)

**Fix Required:**
```typescript
// In runner.ts when watchdog triggers:
import { handleWatchdogTrigger } from '../escalation/triggers';
await handleWatchdogTrigger({ type: 'watchdog', severity: 'critical', ... });

// In orchestrator/executor.ts on workflow failure:
import { handleOrchestrationFailure } from '../escalation/triggers';
await handleOrchestrationFailure(workflowId, error);
```

### 3. Policy Engine Boundary (MEDIUM)

**Problem:** Policy engine exists and works, but isn't called at gateway boundary.

**Impact:**
- Policy evaluation happens later than intended
- Gateway admits events that might be policy-denied

**Fix Required:** Call policy evaluation in `gateway/index.ts` before event processing.
