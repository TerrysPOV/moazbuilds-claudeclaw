**Create skills/update-agent/SKILL.md** (new file). Mirror create-agent's structure and tone.

Frontmatter:
```yaml
---
name: update-agent
description: Use when the user wants to modify an existing agent — change personality, workflow, add or edit a scheduled job, update Discord channels, swap a job's model, or delete the agent. Trigger phrases include "update an agent", "edit agent", "modify agent", "change agent", "/claudeclaw:update-agent", "add a job to <agent>", "remove job from <agent>", "delete agent <name>".
---
```

Body sections:

1. **Intro** — one sentence: this skill walks the user through updating an existing agent without losing its accumulated MEMORY.md state.

2. **Invariants box** (visible to Claude):
   - NEVER read or write `agents/<name>/MEMORY.md` — preserve accumulated state
   - NEVER touch `agents/<name>/session.json`
   - The ONLY exception is the "Delete agent" menu option which removes the entire dir
   - All edits are full-file rewrites via the helpers in src/agents.ts (no offset edits)

3. **Flow**:

   a) **List agents**:
   ```bash
   bun -e 'import { listAgents } from "./src/agents"; console.log((await listAgents()).join("\n"));'
   ```

   b) **Pick agent** (or use $ARGUMENTS if provided).

   c) **Show current state**:
   ```bash
   bun -e '
   import { loadAgent, listAgentJobs } from "./src/agents";
   const ctx = await loadAgent("AGENT_NAME");
   const jobs = await listAgentJobs("AGENT_NAME");
   console.log("SOUL.md (head):"); console.log((await Bun.file(ctx.soulPath).text()).split("\n").slice(0,30).join("\n"));
   console.log("\nJobs:"); for (const j of jobs) console.log(`  - ${j.label} (${j.cron}) ${j.enabled ? "" : "[disabled]"} ${j.model ? "model="+j.model : ""}`);
   '
   ```

   d) **Menu** (one selection at a time, loop until user picks Done):
   1. Workflow → re-prompt full workflow text → `updateAgent(name, { workflow: ... })`
   2. Personality → re-prompt → `updateAgent(name, { personality: ... })`
   3. Add job → collect label, cron, trigger, model → `addJob(...)`
   4. Edit job → list jobs, pick one, prompt for new cron/trigger/enabled/model → `updateJob(...)`
   5. Remove job → list jobs, pick one, **confirm**, then `removeJob(...)`
   6. Discord channels → re-prompt comma-separated → `updateAgent(name, { discordChannels: [...] })`
   7. Data sources → re-prompt → `updateAgent(name, { dataSources: ... })`
   8. **Delete agent** → require user to **re-type the agent name verbatim** as confirmation → `deleteAgent(name)` (this IS allowed to remove MEMORY.md because the agent is going away)
   9. Done → exit the loop

   For each menu action provide an example `bun -e` invocation. Use temp JSON files for multi-line content.

4. **Examples** — show two end-to-end transcripts:
   - "Update Reg's workflow"
   - "Add a new job to Suzy"
